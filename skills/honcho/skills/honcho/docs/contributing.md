# Contributing Guidelines

## Getting Started

1. Set up your development environment
2. Join the community via Discord
3. Review existing issues before beginning work

## Contribution Workflow

1. Fork and clone the repository
2. Create a branch following naming conventions:
   - `feature/`, `fix/`, `docs/`, `refactor/`, `test/`
3. Make changes adhering to coding standards
4. Commit using conventional commit format: `type(scope): description`
5. Submit a pull request with clear descriptions

## Commit Types

`feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`

## Coding Standards

- Follow PEP 8 style guidelines (Python)
- Use type hints where possible
- Keep functions focused and single-purpose
- Use meaningful variable and function names

## Testing

- Write unit tests for new functionality
- Ensure existing tests pass before submitting

## Review Process

1. Automated checks
2. Maintainer code review
3. Discussion/iteration
4. Approval and merging

## Accepted Contributions

Bug fixes, new features, documentation improvements, tests, performance enhancements, and tutorials.

## Support Channels

- **Discord**: General questions
- **GitHub Issues**: Bugs and feature requests
- **Email**: Security issues (private)
