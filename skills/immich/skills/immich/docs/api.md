# API


Immich uses the <a href="https://swagger.io/specification/" target="_blank" rel="noopener noreferrer">OpenAPI</a> standard to generate API documentation. To view the published docs see <a href="https://api.immich.app/" target="_blank" rel="noopener noreferrer">here</a>.

## Generator<a href="#generator" class="hash-link" aria-label="Direct link to Generator" translate="no" title="Direct link to Generator">​</a>

OpenAPI is used to generate the client (Typescript, Dart) SDK. `openapi-generator-cli` can be installed <a href="https://openapi-generator.tech/docs/installation/" target="_blank" rel="noopener noreferrer">here</a>. The generated SDK is based on the `immich-openapi-specs.json` file, which is autogenerated by the server **when running in development mode**. The `immich-openapi-specs.json` file can be modified with `@nestjs/swagger` decorators used or referenced by controller endpoints. See the <a href="https://docs.nestjs.com/openapi/types-and-parameters" target="_blank" rel="noopener noreferrer">NestJS OpenAPI docs</a> for more info. When you add a new endpoint or modify an existing one, you must run the server in development mode and run the command below to update the client SDK.


``` prism-code
make open-api
```


You can find the generated client SDK in the `open-api/typescript-sdk/client` for Typescript SDK and `mobile/openapi` for Dart SDK.
